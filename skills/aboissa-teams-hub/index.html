01 — O QUE É

Não é um chatbot.
É um especialista configurado
para sua empresa.

Aboissa Teams é infraestrutura de IA — 5 Projects especializados que resolvem análise tática, comunicação, financeiro, documentação e design.

Sem padrão: cada pessoa usa IA do seu jeito. Respostas inconsistentes. Análises que não conversam. Tempo perdido em setup.

Com Aboissa Teams: IA funciona como departamento. Cada Project conhece o contexto, segue as regras, entrega análise acionável.

05 — PERGUNTAS COMUNS

Suas dúvidas
respondidas

O que é Claude Teams exatamente? +
Claude Teams é uma infraestrutura de IA colaborativa onde cada Project é um especialista configurado para um contexto específico. No caso da Aboissa, você tem 5 Projects: um para análise tática (Muriel), um para comunicação (Michel), um para financeiro, um para documentação técnica, e outro para design system. Cada um trabalha com padrão consistente e conhece as regras da empresa.
Quanto tempo leva para implementar? +
O setup segue um protocolo de 7 dias: Dias 1-2 você cria os Projects e define as arquiteturas. Dias 3-4 você carrega contexto e documentação. Dias 5-6 você testa e ajusta. Dia 7 você treina o time e ativa em produção. É rápido porque cada fase tem entregáveis claros — sem improviso.
Como funciona a separação entre Muriel e Michel? +
Muriel é análise tática interna — dados sensíveis de spread, margem, estratégia spot. Michel é comunicação externa — traduz essa inteligência para clareza estratégica, sem expor dados sensíveis. Michel SEMPRE deriva de Muriel: você não publica nada que Muriel não descobriu primeiro. Isso protege dados e mantém consistência.
Preciso treinar o time antes de usar? +
Sim, mas o playbook inclui guia de onboarding de primeira semana. Você treina o time durante o setup — cada pessoa aprende seu Project enquanto ele está sendo criado. No Dia 7, quando ativa em produção, o time já sabe como usar. Não é treinamento separado — é integrado ao processo.
O que acontece com a documentação técnica? +
O Project DOCUMENTACAO_TECNICA é seu repositório vivo de specs, runbooks, SLAs, arquitetura. Quando você faz mudança operacional, documenta lá. Quando alguém nova entra, consulta lá. É referência central — mantém todo mundo alinhado. Sem documentação dispersa em Slack ou email.
E se eu já tiver um Radar fazendo análise manual? +
Perfeito — você ainda precisa de Muriel. A diferença é que agora Muriel (Claude Teams) torna a análise consistente, rápida e documentada. Seu Radar manual vira referência — Muriel aprende com o histórico e gera análises 2-5x mais rápido. Ninguém é dispensado — o time fica mais potente.
